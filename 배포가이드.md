# 배포 가이드 — Secret Scanner

**구성:** Render(단일 Docker 컨테이너: nginx + Express) + Neon(PostgreSQL) + GitHub Actions(CI/CD)

> 코드/설정은 이미 repo에 준비됨 (`deploy/Dockerfile`, `render.yaml`, `.github/workflows/ci.yml`).
> 아래는 **계정 연결 단계**(대시보드 클릭)다. 한 번만 하면 이후엔 push 시 자동 배포된다.

---

## 1단계. Neon — PostgreSQL 생성 (무료)

1. https://neon.tech 접속 → **GitHub 계정으로 가입**
2. **Create Project** (이름: `secret-scanner`, 리전은 가까운 곳)
3. 생성 후 **Connection string** 복사
   - 형식: `postgresql://USER:PASSWORD@ep-xxxx.region.aws.neon.tech/dbname?sslmode=require`
   - `sslmode=require`가 포함된 **Pooled connection** 문자열을 쓰면 된다. (앱이 SSL 자동 처리)

---

## 2단계. Render — 배포 (무료)

### 방법 A. Blueprint (render.yaml 자동 인식) — 권장
1. https://render.com 접속 → **GitHub로 가입/로그인**
2. **New → Blueprint** → `JebeenLee/secret-scanner` 저장소 선택
3. Render가 `render.yaml`을 읽어 `secret-scanner` 웹 서비스를 잡는다.
4. **환경변수 `DATABASE_URL`** 에 1단계의 Neon 연결 문자열 입력
5. **Apply / Deploy** → 빌드 시작 (`deploy/Dockerfile` 사용)

### 방법 B. 수동 (Blueprint이 안 잡힐 때)
1. **New → Web Service** → 저장소 선택
2. 설정값:
   - **Runtime/Environment**: Docker
   - **Dockerfile Path**: `deploy/Dockerfile`
   - **Docker Context**: `.` (루트)
   - **Plan**: Free
3. **Environment** 탭 → `DATABASE_URL` = Neon 연결 문자열 추가
4. **Create Web Service**

> 배포가 끝나면 `https://secret-scanner-xxxx.onrender.com` 형태의 URL이 생긴다. **이게 제출용 URL.**

---

## 3단계. GitHub Actions — CI/CD 연결

push할 때마다 GitHub Actions가 빌드 검사 후 Render 배포를 트리거하도록 한다.

1. Render 서비스 → **Settings → Deploy Hook** 의 URL 복사
2. GitHub 저장소 → **Settings → Secrets and variables → Actions → New repository secret**
   - Name: `RENDER_DEPLOY_HOOK`
   - Secret: 복사한 Deploy Hook URL
3. 끝. 이제 `main`에 push하면:
   - `build` 잡: 프론트 빌드 + 백엔드 설치 검사
   - `deploy` 잡: Render Deploy Hook 호출 → 자동 배포

> 시크릿을 설정 안 해도 Render의 기본 자동배포(저장소 연결 시)가 동작한다. Deploy Hook은 "GitHub Actions로 배포를 제어"하는 용도.

---

## 4단계. 확인

- 배포 URL 접속 → **예제 체험** 버튼 → 탐지 결과 + 해설 카드
- `https://<배포URL>/api/health` → `{"status":"ok","db":"on", ...}` (db가 `on`이면 Neon 연결 성공)
- 스캔 몇 번 → "전체 통계" 막대가 쌓이고, 재배포해도 유지되면 DB 영구저장 정상

---

## 트러블슈팅

| 증상 | 원인 / 해결 |
|---|---|
| `/api/health`의 db가 `memory` | `DATABASE_URL` 미설정 또는 오타 → Render 환경변수 확인 |
| DB 연결 SSL 오류 | Neon 문자열에 `?sslmode=require` 포함 확인 (앱이 SSL 자동 처리) |
| 빌드 실패 | Dockerfile Path가 `deploy/Dockerfile`, Context가 `.` 인지 확인 |
| 첫 응답이 느림 | Render 무료 플랜은 유휴 시 슬립 → 첫 요청에 수십 초 (정상) |
