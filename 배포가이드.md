# 배포 가이드 — Secret Scanner

**구성:** Render(단일 Docker 컨테이너: nginx + Express) + Render 관리형 PostgreSQL + GitHub Actions(CI/CD)

> 코드/설정은 이미 repo에 준비됨 (`deploy/Dockerfile`, `render.yaml`, `.github/workflows/ci.yml`).
> `render.yaml`이 **웹 서비스 + PostgreSQL을 함께 생성하고 `DATABASE_URL`을 자동 연결**한다. (DB 따로 가입 X)
> 아래는 **대시보드 클릭 단계**다. 한 번만 하면 이후 push 시 자동 배포된다.

---

## 1단계. 변경분 push

```bash
cd secret-scanner
git add .
git commit -m "chore: Render 관리형 Postgres 로 배포 구성"
git push
```

---

## 2단계. Render — Blueprint로 배포 (무료)

1. https://render.com 접속 → **GitHub로 가입/로그인**
2. **New → Blueprint** → `JebeenLee/secret-scanner` 저장소 선택
3. Render가 `render.yaml`을 읽어 자동으로 생성:
   - **PostgreSQL** `secret-scanner-db` (관리형, 영구저장)
   - **웹 서비스** `secret-scanner` (Docker, `deploy/Dockerfile`)
   - 둘을 연결하는 **`DATABASE_URL` 자동 주입** ← 직접 입력할 거 없음
4. **Apply** → 빌드·배포 시작
5. 끝나면 `https://secret-scanner-xxxx.onrender.com` URL 생성 → **이게 제출용 URL**

---

## 3단계. GitHub Actions — CD 연결 (선택)

push할 때마다 GitHub Actions가 빌드 검사 후 Render 재배포를 트리거하도록 한다.

1. Render의 `secret-scanner` 웹 서비스 → **Settings → Deploy Hook** URL 복사
2. GitHub 저장소 → **Settings → Secrets and variables → Actions → New repository secret**
   - Name: `RENDER_DEPLOY_HOOK`
   - Secret: 복사한 Deploy Hook URL
3. 이후 `main`에 push하면 `deploy` 잡이 실제로 Render 배포를 트리거한다.

> 시크릿을 안 넣어도 Render가 repo 연결 시 자동배포(auto-deploy)를 한다. Deploy Hook은 "GitHub Actions로 배포를 제어"하는 용도.

---

## 4단계. 확인

- 배포 URL 접속 → **예제 체험** 버튼 → 탐지 결과 + 해설 카드
- `https://<배포URL>/api/health` → `{"status":"ok","db":"on", ...}` (db가 `on`이면 DB 연결 성공)
- 스캔 몇 번 → "전체 통계" 막대가 쌓이고, 재배포해도 유지되면 DB 영구저장 정상

---

## 트러블슈팅

| 증상 | 원인 / 해결 |
|---|---|
| `/api/health`의 db가 `memory` | DB 연결 실패 → Render 웹 서비스 환경변수에 `DATABASE_URL`이 잡혔는지 확인 |
| Blueprint이 DB를 안 만듦 | `render.yaml`의 `databases:` 섹션 확인, Blueprint 재적용 |
| 빌드 실패 | Dockerfile Path가 `deploy/Dockerfile`, Context가 `.` 인지 확인 |
| 첫 응답이 느림 | Render 무료 플랜은 유휴 시 슬립 → 첫 요청에 수십 초 (정상) |
| 무료 DB 만료 | Render 무료 Postgres는 약 30일 후 만료 (과제 기간엔 충분) |
